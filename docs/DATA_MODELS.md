# Data Models

## Introduction

The AI Response Quality Evaluator Agent processes different types of data throughout the evaluation pipeline. These data models define the information required as input, the intermediate evaluation results, and the final output generated by the system.

---

# 1. Input Data Model

The evaluation process begins by collecting the following information:

| Field | Description |
|--------|-------------|
| User Question | The question submitted by the user. |
| AI Generated Response | The response generated by the Large Language Model (LLM). |
| Reference Context *(Optional)* | Trusted information retrieved using the RAG pipeline for evaluation. |

### Example

| Field | Value |
|--------|-------|
| User Question | What is Artificial Intelligence? |
| AI Generated Response | Artificial Intelligence is the simulation of human intelligence by machines. |
| Reference Context | Retrieved information from the knowledge base. |

---

# 2. Evaluation Data Model

Each evaluation agent analyzes the response independently and generates its own result.

| Evaluation Parameter | Output |
|----------------------|--------|
| Relevance | Relevance Score & Feedback |
| Accuracy | Accuracy Score & Feedback |
| Hallucination | Hallucination Score & Feedback |
| Completeness | Completeness Score & Feedback |

---

# 3. Final Report Data Model

The Verdict Agent combines the outputs from all evaluation agents to generate the final report.

The report contains:

- Relevance Score
- Accuracy Score
- Hallucination Score
- Completeness Score
- Overall Quality Score
- Final Evaluation Summary

---

# Data Flow

The overall flow of information through the system is shown below.

```text
User Question
        │
        ▼
Large Language Model
        │
        ▼
AI Generated Response
        │
        ▼
Evaluation Agents
        │
        ▼
Evaluation Scores
        │
        ▼
Final Evaluation Report
```

---

# Summary

The data models provide a structured way of passing information between different modules of the system. Separating input data, evaluation results, and final reports keeps the evaluation pipeline organized and simplifies future enhancements.
